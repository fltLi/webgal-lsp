# 代码诊断

## 简介

`webgal-language-service` 提供对 WebGAL 脚本全面的代码诊断功能。

当**场景或资源发生更改时**将为**所有场景文件重新推送**一遍诊断。

诊断分为四个级别：`ERROR`，`WARNING`，`INFO`，`HINT`。  
仅当前两个级别的诊断全部被消除或忽略后，才会显示 `INFO` 级别的诊断。

考虑到忽略诊断的需求，`webgal-language-service` 引入了一项新的 `nolint` 语法。  
通过在**语句注释开头追加 `nolint` 参数**，实现对指定语句和错误码的忽略：
```txt
lsp:我将推送诊断;nolint:WG002|WG001|WG004; <-执行过滤。
```

## 类型

### 语法检查

检查每条语句自身的**语法及格式问题**，与上下文无关。

此功能由 `webgal-language-core` 提供，诊断伴随语句解析在编辑场景时**文件级更新**。

| 错误码  | 级别      | 描述                                                                             |
| ------- | --------- | -------------------------------------------------------------------------------- |
| `WG001` | `INFO`    | 语句格式不规范，如多余空格、参数顺序不当、未采用语法糖、存在未知与重复的参数等。 |
| `WG002` | `ERROR`   | 语句主参数或参数类型错误。                                                       |
| `WG003` | `WARNING` | 设置重复或相互冲突的语句参数。                                                   |
| `WG004` | `ERROR`   | 语句参数缺少实现其功能所需的其他参数。                                           |
| `WG005` | `WARNING` | 设置已弃用或不建议使用的语句参数。                                               |
| `WG006` | `WARNING` | 未知或无法识别的语句参数。                                                       |
| `WG008` | `WARNING` | 条件执行的 `-when` 表达式结果为常量。                                            |

### 环境诊断

检查每条**语句所依赖的资源**、全局状态等是否存在问题。

此功能由 `webgal-language-service` 提供，每次推送诊断前将扫描**全部场景**以构建诊断信息。

| 错误码  | 级别    | 描述                             |
| ------- | ------- | -------------------------------- |
| `WG007` | `ERROR` | 语句依赖的资源不存在或无法识别。 |

### 逻辑分析

通过**模拟脚本运行**分析代码的逻辑问题。

此功能由 `webgal-simulate` 提供，每次推送诊断时将针对触发项目执行**全量模拟**，并构建诊断信息。

| 错误码  | 级别      | 描述                                                     |
| ------- | --------- | -------------------------------------------------------- |
| `WG009` | `HINT`    | 模拟执行中断，如达到最大限制、遭遇 `random()` 表达式等。 |
| `WG010` | `WARNING` | 语句（或场景）不可达。                                   |
| `WG011` | `ERROR`   | 语句依赖的变量、标签、舞台对象未定义。                   |
| `WG012` | `WARNING` | 设置执行效果与上文重复的无意义语句。                     |
| `WG013` | `WARNING` | 连续执行块以 `wait` 语句结尾，被迫打断。                 |

#### 设计

模拟执行的核心目标是在不实际运行游戏的情况下，尽可能还原脚本的执行，检测静态分析无法捕获的动态逻辑问题。

##### 核心概念

- **舞台状态**  
  维护当前场景的**完整舞台信息**，包括：舞台对象及其效果（如背景、立绘、音效、特效等），状态增量，局部和全局变量，场景调用栈。

- **状态增量**  
  每条语句产生一个增量，描述该语句对舞台或变量表的影响。  
  增量计算完成后，若当前语句非连续执行，将应用增量以更新舞台状态。

- **检查点**  
  为防止模拟陷入无限循环，在以下位置设立检查点：
  - **跳转目标**：每条场景的开头，以及 `label` 语句所指定的标签处。
  - **条件执行**：带有 `-when` 的条件执行语句。
  - **变量定义**：使用 `setVar` 和 `getUserInput` 定义或修改变量的语句。

  每次经过检查点时，模拟器计算当前舞台状态的 `u64` 指纹并记录。  
  若该状态已存在，或同一检查点的历史状态超过上限（`32` 次），将中断当前分支的模拟（`WG009`）。

  通过横向剪枝，检查点机制将模拟**复杂度控制在线性**级别。

##### 分支模拟

`webgal-simulate` 支持多分支模拟，其采用 DFS 的方式分别遍历。

- **`choose`**  
  触发分支选择时，为每个选项克隆当前状态并分别继续模拟，所有分支产生的诊断合并后统一推送。

- **`getUserInput`**  
  支持通过 `-lintValues=["输入1", "输入2"]` 参数传入预设输入值，每个值视为一个独立分支进行模拟。  
  若未提供该参数，则仅记录变量为默认值（不存在时为空字符串）。

  存在正则校验且未通过时，将触发 `WG003` 报错，并中断当前分支的模拟（`WG009`）。

- **`random()`**  
  随机值无法确定，模拟器分别代入区间两端和中点进行分支探索。  
  若未来 WebGAL 原生支持预设随机值嵌入，再考虑调整此策略。

##### TODO：并发模拟

未来版本将引入多线程并行模拟架构以提升大型项目的诊断性能。

模拟被拆分为两个阶段：

- **引导线程**
  负责轻量级遍历，仅处理变量追踪、条件执行（`-when`）、调用栈维护及舞台对象的浅层状态（仅标记存在性，不维护具体内容）。

- **计算线程池**
  利用视觉小说的“转场”特性，当舞台对象全部清空时，引导线程在此处设立栅栏，将项目切割为独立块。  
  每个块连同变量状态一并分发至线程池，并行执行全量舞台状态计算与详细诊断。
